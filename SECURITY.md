# Security Policy

## Supported Disclosure Path

If you believe you have found a security vulnerability in Synaply, please do not open a public GitHub issue.

Preferred reporting path:

1. Use GitHub private vulnerability reporting if it is enabled for the repository.
2. If private reporting is not available yet, contact the repository owner privately before any public disclosure.

## What to Include

Please include:

- a short description of the issue
- affected area or file path
- reproduction steps
- potential impact
- any suggested mitigation if you have one

## Disclosure Expectations

- Please give the maintainers a reasonable amount of time to investigate and prepare a fix before public disclosure.
- If the report is confirmed, the maintainers will aim to acknowledge it, assess impact, and coordinate a fix before publishing details.

## Scope

This policy applies to:

- application code in this repository
- local Supabase configuration committed with the project
- deployment and environment handling documented in the repository

It does not grant permission for destructive testing against third-party infrastructure or services.
